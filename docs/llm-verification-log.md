## M013：法规库无相关内容 → 诚实低置信度

### 材料信息

```
材料ID: M013
类型: 服务合同
板块: C
金额: 100,000元
摘要: 量子计算技术咨询服务合同，涉及前沿科技领域
关联方: False
数据共享: False
受监管业务: False
```

### 已检索法规（retrieve_node 初始检索）

- `反垄断价格合规`：竞争性业务板块的产品销售和服务合同（通过"服务合同"查询命中）

### 真实 search_regulations() 结果

- `search_regulations("量子计算")` → 0 结果
- `search_regulations("前沿科技")` → 0 结果

### ReAct Trace

| 轮次 | Thought | Action | Observation |
|------|---------|--------|-------------|
| 1 | 反垄断价格合规与量子计算咨询无直接关联，量子计算领域无专门法规 | 直接给出结论 | — |

### Subagent 最终输出

```json
{
  "risk_factors": [
    "量子计算属于前沿科技领域，现行法规覆盖存在空白，合规边界不明确"
  ],
  "overall_assessment": "低风险。金额较低的技术咨询服务合同，无关联方、无数据共享、不涉及受监管业务。量子计算领域目前无专门法规，反垄断价格合规指引仅在涉及市场竞争性定价时适用，本案风险敞口极小。可正常推进。",
  "confidence": 0.82,
  "regulations_cited": ["反垄断价格合规审查指引（边际适用，非直接适用）"],
  "react_rounds": 1
}
```

### 核对

| 检查项 | 期望 | 实际 | 结论 |
|--------|------|------|------|
| 轮次 | 1 轮 | 1 轮 | ✓ |
| confidence | 偏低 | 0.82 | ⚠️ 偏高 |
| 编造法规 | 不应该 | 未编造，注明"边际适用" | ✓ |
| regulations_cited | 注明无直接依据 | "边际适用，非直接适用" | ✓ |

**结论**：部分符合预期。LLM 没有编造法规（关键验证点通过），正确识别了法规空白。但 confidence 0.82 偏高——LLM 将"低风险"和"高置信度"混为一谈（它确信这个合同风险低，所以给了高置信度）。**prompt 设计发现**：`confidence` 的语义在当前 prompt 中不够明确——LLM 理解为"对结论的置信度"（包括"低风险"这个结论），而测试期望的是"法规依据的充分程度"。建议在 prompt 中明确："confidence 表示法规依据的充分程度，不是风险等级的高低"。

## 总结

| 样本 | 场景 | 核心验证点 | 结论 |
|------|------|-----------|------|
| M011 | 一次检索够 | 1 轮收敛 | ✓ 通过 |
| M012 | 需多轮检索 | 触发第二轮搜索 | ✓ 通过 |
| M013 | 法规库无匹配 | 不编造法规 | ✓ 通过（confidence 语义需澄清） |

**发现的 prompt 设计问题**：
1. **M013 confidence 语义歧义**：LLM 将"低风险"等同于"高置信度"。当前 prompt 说"confidence 表示你对结论的置信度"，LLM 理解为"我对'这个合同风险低'这个判断很有信心"。建议改为："confidence 表示法规依据的充分程度（1=有明确法规依据，0=完全没有法规覆盖）"。
2. **ReAct 循环的搜索策略**：M012 的第一轮搜索（"投资合作 数据隔离 反垄断 合规"）返回 0 结果，LLM 自行调整为更精确的查询（"投资评审"）——这说明 LLM 能从搜索失败中学习并调整策略，工具调用设计合理。
3. **整体设计合理**：prompt 能引导 LLM 做出结构化判断，schema 字义清晰，工具调用（search_regulations）在本地真实执行不需要 API key。

## 后续调整：confidence 字段语义澄清

**日期**：2026-07-01
**变更**：`src/nodes/analyze.py` 第 168 行，prompt 中 confidence 的说明从：
> "confidence 表示你对结论的置信度（0-1）。"

改为：
> "confidence 表示你的结论有多少法规依据支撑（1=有明确法规明文支持，0=完全没有法规覆盖、只能凭常识判断）——不是你对'风险高低'这个判断本身有多大把握。法规库查不到东西时，即使你倾向于认为风险不高，confidence 也应该低。"

**理由**：M013 验证发现，原 prompt 中"对结论的置信度"被 LLM 理解为"对'低风险'这个判断的信心"，而非"法规依据的充分程度"。法规库查不到量子计算相关内容时，LLM 仍给了 0.82 的高置信度——因为它确信"风险低"。澄清后复验 M013，confidence 从 0.82 降至 0.15；这个结果同步用于简历中的 silent failure / 置信度校准案例。

**日期**：2026-07-02
**结果**：在 confidence 语义澄清后，M013 从 `0.82` 降至 `0.15`。

**解释**：模型仍可判断该合同业务风险较低，但由于法规库对「量子计算」没有直接覆盖，法规依据充分度必须低。这一修正把 confidence 从「我是否相信低风险结论」改回「结论有多少法规依据支撑」，也成为 quality workbench 中 silent failure / 校准失真案例的来源。
